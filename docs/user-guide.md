# VolunTrack User Guide

## Overview

VolunTrack is a desktop volunteer management application that allows users to discover and register for community projects. The application supports two types of users:

* **Volunteers** can create accounts, browse available projects, register for opportunities, and track their participation history.
* **Administrators** can manage community projects, monitor registrations, and maintain project availability.

---

# Getting Started

## Creating a Volunteer Account

New volunteers can create an account through the registration page.

The registration process requires:

* Full name
* Username
* Email address
* Password
* Password confirmation

The system validates the provided information before creating the account.

Once registration is complete, users can log in using their username and password.

---

## Logging In

Users can access the application through the login screen.

After successful authentication:

* Volunteers are directed to the volunteer dashboard.
* Administrators are directed to the administrator dashboard.

---

# Volunteer Workflow

## Browsing Available Projects

The volunteer dashboard displays available community projects.

Users can view project information such as:

* Project name
* Location
* Available day
* Hourly value
* Available slots

Volunteers can select projects that they are interested in joining.

---

## Registering for Projects

Selected projects are added to the user's registration cart.

Before confirming a registration, volunteers can:

* Review selected projects
* Adjust the number of volunteer slots
* Modify volunteer hours

After reviewing their selections, users can proceed with registration confirmation.

---

## Registration Confirmation

VolunTrack includes a simulated email verification process to confirm registrations.

During confirmation:

1. A verification code is generated by the system.
2. The generated code is displayed within the application to simulate an email delivery process.
3. Users enter the verification code to complete registration.

Once verified, the registration is saved to the database.

---

## Viewing Participation History

Volunteers can view their previous and current project registrations through the participation history page.

Displayed information includes:

* Registration ID
* Project name
* Location
* Scheduled day
* Number of slots
* Volunteer hours
* Contribution value
* Confirmation timestamp

---

## Exporting Participation History

Users can export their participation history as a CSV file.

The exported file contains:

* Registration ID
* Project name
* Location
* Day
* Number of slots
* Volunteer hours
* Contribution value
* Confirmation date and time

The exported CSV file is saved directly within the VolunTrack project directory.

Example export format:

```csv
RegID,Project,Location,Day,Slots,Hours,Contribution,ConfirmedAt
0002,Food Drive,Community Hall,Sun,1,2,70.0,2026-07-12T22:14:26.814444300
```

# Administrator Workflow

## Accessing the Administrator Dashboard

Administrators can access additional management functionality after logging in with an administrator account.

The administrator dashboard provides access to:

* Project management
* Registration management
* Volunteer participation records

---

## Managing Projects

Administrators can:

* Create new volunteer projects
* Edit existing project information
* Adjust project capacity
* Enable or disable project availability

Disabled projects will no longer appear as available options for volunteers.

---

## Viewing Volunteer Registrations

Administrators can view registration information across all volunteers.

This allows administrators to monitor:

* Registered projects
* Volunteer participation
* Project capacity
* Confirmation records

---
